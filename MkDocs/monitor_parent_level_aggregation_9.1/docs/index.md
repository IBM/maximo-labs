# Welcome to the Maximo Monitor 9.1</br>Parent-Level Aggregation Lab

!!! info
    This lab demonstrates how to use **Parent-Level Aggregation**, a powerful charting capability introduced in Maximo Monitor 9.1.

Parent-Level Aggregation enables you to visualize child resource metrics directly on parent resource dashboards.
</br>

### Key Benefits of Parent-Level Aggregation

* Build comprehensive dashboards by aggregating child resource data at the parent level
* Apply aggregation functions (sum, average, min, max, count) on child metrics
* Configure time-based grain (hourly, daily, weekly, monthly) for child resource data
* Visualize device data from multiple child resources in a single parent dashboard

### Supported Resource Types

Hierarchy dashboards with parent-level aggregation are available for the following resource types:

* **Asset** - Aggregate data from devices assigned to assets
* **Location** - Aggregate data from assets and devices assigned to locations
* **System** - Aggregate data from locations, assets and devices within systems
* **Site** - Aggregate data systems, locations, assets, and devices within sites
* **Organization** - Aggregate data across the entire organization hierarchy

### Lab Focus

!!! tip
    In this lab, we will use **Asset** as the parent resource to demonstrate parent-level aggregation. However, the same concepts and steps apply to all hierarchy levels including Location, System, Site, and Organization.

### What You Will Learn

In this lab, you will learn how to:

* Add child resource data items to parent resource dashboards (using Asset as an example)
* Configure aggregation functions (sum, average, min, max, count)
* Create charts visualizing aggregated child resource data
* Build comprehensive hierarchy dashboards without data duplication
