MAS Monitor supports CSV-based data ingestion as easily accessible alternative to real-timestreaming protocols like MQTT or HTTP. 

This approach provides organizations with a straightforwardmethod to implement without the complexity of establishing continuous data connections.

CSV-based ingestion is a flexible, low-barrier alternative to real-time streaming protocols (MQTT/HTTP)for Asset Performance Management (APM) and Condition-Based Maintenance (CBM).

CSV ingestion bridges this gap by allowing external systems to export time-series data in standardizedCSV formats, which Monitor can periodically ingest from accessible storage locations (IBM cloudstorage, AWS S3 buckets or Persistent Volume).
